# Rattin

Torrent-based media streaming app with a Netflix-style TMDB browse UI and built-in video player. Key differentiator: phone remote control — scan a QR code to pair a phone as a remote for the PC player via SSE relay.

## Stack

- **Backend:** Single-file Express 5 server (`server.js`, ~1800 lines), Node.js, ES modules
- **Frontend:** React 19 + React Router 7, Vite 6, plain CSS (no Tailwind, no CSS modules)
- **Torrent engine:** WebTorrent (in-process, server-side)
- **Transcoding:** ffmpeg/ffprobe spawned as child processes
- **Metadata:** TMDB API with in-memory TTL cache (`lib/cache.js`)
- **Real-time:** Server-Sent Events for remote control relay (no WebSocket)
- **Auth:** nginx basic auth in prod, bypassed via cookie token; phone remotes get a separate auth token
- **DB:** None. All state is in-memory (Maps, Sets). Playback position saved in browser sessionStorage.

## Folder structure

```
server.js            # ALL backend logic — Express routes, torrent management, transcode, SSE relay
lib/cache.js         # TTLCache class + TMDB fetch helper (imported by server.js)
lib/seek-index.js    # Keyframe index builder for seeking in incomplete torrents
src/                 # React frontend (Vite dev server proxies /api to :3000)
  lib/PlayerContext.jsx  # Global player state, video element, RC session SSE
  lib/api.js             # All fetch wrappers for /api/* endpoints
  lib/utils.js           # formatTime and similar helpers
  pages/                 # Route-level components (Home, Detail, Player, Search, Remote)
  components/            # Shared UI (Navbar, MiniPlayer, MovieCard, ContentRow, etc.)
public/              # Vite build output (gitignored except structure)
deploy/              # Ansible playbook for prod deployment
```

## How to run

```bash
cp .env.example .env   # Add your TMDB_API_KEY
npm install
npm start              # Backend on :3000 (serves built frontend from public/)
npm run dev            # Vite dev server on :5173 (proxies /api -> :3000)
npm run build          # Build frontend into public/
```

IMPORTANT: `npm start` uses `node --env-file=.env` (Node 20+). No dotenv package.

ffmpeg and ffprobe must be installed on the system for transcoding and media probing.

## Architecture decisions

- **Single server.js:** All backend logic lives in one file. Do not split into route files or controllers without explicit instruction. Add new routes in the same file following the existing pattern.
- **No database:** Torrent state, transcode jobs, and RC sessions are all in-memory Maps. This is intentional — the app is single-user/single-machine.
- **SSE instead of WebSocket:** Remote control uses Server-Sent Events (`/api/rc/events`) with REST POST for commands. The player reports state every 1s; the remote sends commands via POST. Do not introduce WebSocket.
- **Transcode strategy:** Non-browser-native formats (anything not .mp4/.m4v/.webm) get transcoded via ffmpeg. Tries remux first (fast, copy streams), falls back to full H.264 re-encode.
- **Seek index:** For incomplete torrents, `lib/seek-index.js` builds a keyframe-to-byte-offset map so the server can prioritize downloading the right torrent pieces for seek operations.
- **Vite build output goes to `public/`:** `publicDir: false` in vite config. The Express server serves `public/` as static files. The built `public/index.html` is gitignored.
- **Stale-while-revalidate caching:** TMDB endpoints use `getStale()` for trending/discover/search — serve expired cache immediately, refresh in background.

## Code style

- ES modules everywhere (`import`/`export`, `"type": "module"` in package.json). Do not use `require()`.
- React components use `.jsx` extension, plain JS files use `.js`.
- Functional components only (no class components). Hooks for all state.
- CSS is per-component plain CSS files imported directly (e.g., `import "./Remote.css"`). Do not introduce CSS modules or styled-components.
- Backend logging uses the `log(level, msg, data)` helper in server.js. Do not use `console.log` directly.
- Frontend API calls go through `src/lib/api.js` wrappers. Do not call `fetch("/api/...")` directly from components.
- The `PlayerContext` is the single source of truth for playback state. Components access it via `usePlayer()` hook.
- No TypeScript. Do not add type annotations or `.ts`/`.tsx` files.

## What NOT to touch

- `public/index.html` and `public/assets/` — these are Vite build artifacts, regenerated by `npm run build`
- `deploy/inventory.ini` — contains prod server credentials (gitignored)
- `.env` — contains TMDB API key (gitignored)
- `AGENT_PROMPT_CACHE.md` — historical prompt, not part of the app
- Do not install new npm packages without explicit approval. The dependency footprint is intentionally minimal.
